🎯 Topic Deep Dives — Shortcuts & Patterns
🔴 SQL / Databases
14–16 marks ▶
⚡ Quick Method — The SELECT Template Every SQL question in all 4 years follows this exact skeleton:

SELECT [col / AGG(col)] FROM [table] WHERE [condition] GROUP BY [col] HAVING [agg condition] ORDER BY [col] ASC/DESC;

Learn to plug & play into this template. 90% of marks come from just knowing which clause goes where.
  • Aggregate shortcuts: COUNT(*) = all rows, COUNT(col) = non-null, AVG/SUM/MAX/MIN — always paired with GROUP BY in 5-mark questions.
  • WHERE vs HAVING rule: WHERE filters rows BEFORE grouping. HAVING filters AFTER. The exam asks this every year either as MCQ or short answer.
  • JOIN pattern: Appears in Section C. Always: WHERE table1.col = table2.col — no fancy syntax needed.
  • DDL you must know: CREATE TABLE, ALTER TABLE ADD/DROP/MODIFY, DROP TABLE, DESC tablename (show structure).
  • NULL trap: Use IS NULL / IS NOT NULL — never = NULL. This is a guaranteed MCQ or correction question.
  • DISTINCT shortcut: Always asked. SELECT DISTINCT col FROM table eliminates duplicates.
  • Primary vs Foreign key: FK is the PK of another table referenced here. Appears in Assertion-Reason and fill-in-the-blank every year.
  • Degree & Cardinality: Degree = number of columns, Cardinality = number of rows. Appears in Section E guaranteed.
  • SHOW TABLES: Write this to display tables in a database — 1 mark guaranteed in Section B or C.

🔴 Python — Output Tracing
8–12 marks ▶
⚡ Feynman Method — Trace Like a Computer For every output question: Write variable values in a table as you go line by line. Never try to "see" the output mentally. Write it out. Takes 2 extra minutes, saves 2 marks per question.
  • Global variable trap: If global x is inside a function, it modifies the outer variable. Without it, the local copy changes only. Appears in 3/4 years.
  • Default argument: def f(a, b=10) — if called with one argument, b=10 is used. If called as f(x=5), keyword argument overrides.
  • String methods tested: .upper(), .lower(), .isupper(), .split(), .upper() — always appear in 2-mark output questions.
  • List mutation in functions: Lists are passed by reference — changes inside function affect original. Integers are NOT.
  • while loop with decrement: Track the loop counter carefully. The print happens before or after decrement — read the order.
  • return multiple values: return a, b gives a tuple. Unpacking with x, y = func() is common. Seen in 23-24.
  • isupper()/islower() tracing: On string 'Programmer@@Pascal' — special chars return False for isupper(). Practice this exact pattern.

🔴 File Handling
8–10 marks ▶
⚡ The 3-Type Framework — Memorize Once, Score Every Year ALL file questions are about one of 3 types: Text file (.txt) · Binary file (pickle) · CSV file. Learn the template for each and fill in the blanks — that's literally what Section E asks.

Text File

open("f.txt","r/w/a")

read() → full string
readline() → one line
readlines() → list
write("text")

Binary File

import pickle
open("f.dat","rb/wb/ab")

pickle.dump(obj, f)
pickle.load(f)
try/EOFError to stop

CSV File

import csv
open("f.csv","w/r")

csv.writer(fh)
writer.writerow(list)
csv.reader(fh) → loop
  • File mode MCQ trick: wb+ = write+read binary. The MCQ always tests wb vs wb+. Answer: wb+ for both reading AND writing in binary.
  • EOFError pattern: Every binary read uses try...except EOFError: break. This exact structure appears in 3/4 years as fill-in-the-blank.
  • Text file word count: f.read().split() splits on ALL whitespace. f.read().split('\n') splits on lines. Count with len().
  • Count specific words: Loop through z = f.read().split(), check if i == "My". This pattern appeared in 24-25 and 22-23 BOTH.
  • Section E fill-in pattern: Always: Line1=open for writing, Line2=write/dump, Line3=open for reading, Line4=read/load/print. 4 marks easy.
  • CSV newline fix: open("f.csv","w",newline='') — prevents blank lines. Asked once in 24-25.
🔴 Stacks
5–6 marks ▶
⚡ 4-Line Stack Template — Memorize This Verbatim Every stack question needs PUSH and POP. The template never changes:

The Universal Stack Template

stack = []

PUSH: stack.append(item)
POP:
if stack: return stack.pop()
else: print("Stack Underflow")

DISPLAY: if stack: print(stack) else: print("Empty")
  • Stack LIFO: Last In First Out. MCQ trap — options say FIFO (that's queue). Stack = LIFO always.
  • Stack pointer: Assertion question said "SP increases/decreases by 1 on push/pop" — both A and R true in 23-24.
  • MCQ sequence tracing: push(4),push(7),pop(),push(1),push(3),pop(),push(5) → stack = [4,1,5]. Top = 5. Work step by step.
  • Applications of stack (2-mark theory): Undo feature, recursion call, expression evaluation, browser back button — memorize any 2.
  • Condition questions: "Push elements divisible by 5" — just add if num % 5 == 0: stack.append(num) inside the loop.

Section D 5-Mark Template

(a) Cable layout → Star topology (draw)
(b) LAN device → Switch/Hub
(c) Server location → Highest computers block
(d) Delhi-campus link → Satellite/Microwave
(e) Data security → Firewall/Antivirus
  • LAN vs MAN vs WAN: LAN = same building (meters), MAN = city (km), WAN = countries (1000+ km). Distance is the key.
  • Wireless for long distance: Satellite link for inter-city/international. Optical fibre for local high-speed. This appears every year.
  • Server placement: Always the block with MOST computers. Easy 1-mark question.
  • Topology MCQ: BUS = single cable, all connected. STAR = central switch. RING = loop. MESH = every node connected. Know their advantages/disadvantages.
  • Protocol MCQs: SMTP/POP = email, FTP = file transfer, HTTP = web, TCP/IP = internet communication.
  • Circuit vs Packet switching: Circuit = dedicated path (telephone), Packet = data broken into packets (internet). Asked in 24-25.

⚖️ Assertion-Reasoning — 2 Marks Free (If You Know the Rule)

The Only 4 Answer Choices — Always the Same

(a) Both A and R true, R is correct explanation ← Most common answer
(b) Both A and R true, R is NOT correct explanation
(c) A is true, R is false
(d) A is false, R is true

Pattern from 4 years: Stack LIFO → (a) both true, R explains A. File mode 'a' → (a) both true. Default args → (c) A true, R false. File modes for CSV/TSV → (a) both true.

🏆 Exam Day Tips — Don't Lose Easy Marks
✍️ Writing Code in Exam
  • Always write def functionname(): first, then indent body
  • End file handling with f.close() — 0.5 marks often
  • Use #comments to explain logic — gets partial credit
  • For SQL, end every statement with semicolon ;
  • Write column names in SQL exactly as given in question table
⚡ Section E Strategy (8 marks)
  • Both Q34 and Q35 have fixed patterns — SQL table + file handling fill-in
  • SQL table: (i) DESC command, (ii) degree/cardinality, (iii) ALTER ADD, (iv) primary key
  • File fill-in: always 4 lines — open write, write, open read, read/print
  • Degree = count columns. Cardinality = count rows. Never confuse.
  • Primary key = column with all unique, non-null values — usually the ID column
🎯 Assertion-Reason Quick Win
  • Read Assertion first, decide if TRUE or FALSE
  • Read Reason, decide if TRUE or FALSE
  • If both true: does R actually explain WHY A is true? → (a) or (b)
  • Stack is LIFO (A=true). Reason about SP → (a) both true, R explains A
  • File mode 'a' appends (A=true), reason is correct → (a)
🔢 Output Tracing Method
  • Draw a table: Variable | Value. Update after each line.
  • For while loops: write each iteration's values before deciding output
  • Check if variable is local or global before assigning value
  • String slicing: str[-9:-0:-1] → -0 is treated as 0, so this gives empty! Common trap.
  • print(end="") — means print on same line with no separator
